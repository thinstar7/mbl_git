요소별 작성 규칙
[1] 속성 작성 순서

class속성과 style속성은 가장 뒤에 작성함. 속성의 작성 순서는 다음의 코드예를 참고.

<input type="text" id="user_id" title="사용자ID" class="input_txt" style="width:100px" />
[2] <title>

현재의 서비스 화면명 또는 콘텐츠명이 가장 먼저 기술됨.

[3] 테이블 요소
1. <table>

summary 애트리뷰트로 표의 요약 내용을 표기.

[표기예] <table summary="보험상품을 주계약과 특약사항의 항목으로 구성하여 설명함.">

2. <caption>

caption 요소는 선언하여 표의 제목을 표현.

[표기예] <caption>보험상품 안내</caption>

3. <colgroup>과 <col>

필요시 선택 사용, <col>의 width는 style애트리뷰트를 사용하여 설정함.

[표기예] <colgroup><col style="width:100px" /></colgroup>

4. <thead>

표 머리글을 그루핑할 때 선언

5. <tfoot>

표 바닥글을 그루핑할 때 선택적으로 선언하고 thead와 tbody 요소 사이에 위치함.

6. <th>

scope 애트리뷰트는 현재의 셀이 어느 셀의 제목인지 범위를 명시하므로 반드시 선언함.(id, headers 사용시 사용하지 않음.) scope 애트리뷰트의 값으로는 col, colgroup, row, rowgroup이 있고 적절히 구분해서 사용함.

[4] 폼 요소
1. <legend>

<fieldset> 사용시 폼 컨트롤 요소들의 그룹명인 <legend>을 반드시 선언.

2. <input>
  • ① 동일한 스타일의 input요소의 텍스트, 너비값을 다르게 설정 해야 하는경우 style애트리뷰트를 이용해서 width를 설정함.
  • ② readonly="readonly"와 기본 선택 표현시, checked="checked"의 표준표기법 사용.
  • ③ type이 image인 경우 alt속성을 반드시 선언.
3. <select>
  • ① 동일한 스타일의 select요소의 너비값을 다르게 설정 해야 하는경우 style애트리뷰트를 이용해서 width를 설정함.
  • ② selected="selected"의 표준표기법 사용.
4. <label>
  • ① <input>(text, checkbox, radio, file, password), <select>, <textarea>의 온라인 서식의 id와 <label> 요소의 for속성을 1:1매칭하는 것을 기본원칙으로 함.
  • ② 명시적인 제공을 기본원칙으로 함.
  • ③ 전화번호와 같이 입력 양식이 1개 2개 이상인 경우, 가장 앞의 요소에만 label의 for속성과 id를 매칭함. 가장 앞의 요소를 포함하여 title속성을 추가해서 접근성 처리를 함.
  • ④ <select>, <textarea>에 연결할 label요소가 위치 할 수 없는 경우에는 title속성을 추가함.
5. <button>

동적 기능 제어를 위해 포커스 요소를 마크업 할 경우, 사용에 따라 <button>과 <a>를 적절히 구분하여 병행 사용.

[5] 기타 요소
1. <h*> (헤딩)

서비스 화면 전체 내용의 흐름으로 보아, 타이틀성인 본문 또는 단락의 제목에 순차 사용.

2. <a>
  • ① 새창이동시 target="_blank" title="새창열림" 추가.
  • ② 문법에 따라 <a>요소 안에 블록 속성의 요소를 포함할 수 없음.
3. <img>와 <area>

alt속성은 반드시 선언해야함.

4. <iframe>

스크린 리더 사용자를 위해 title 애트리뷰트 반드시 추가.

기본규칙
  • 특정 브라우저 또는 버전에서 최적화하기 위한 Hack의 사용은 지양.
  • 속성은 영문 소문자로만 작성
  • 공백이 포함된 폰트명, 한글 폰트명, 문자열 데이터 타입, filter 속성의 파라미터값은 작은따옴표(' ')로 감싼다.
  • url 데이터 타입에는 작은 따옴표를 사용하지 않는다.
  • @charset 선언 시에는 속성 값을 큰따옴표(" ")로 감싼다.
font-family: '돋움'

filter:progid:DXImageTransform.Microsoft.AlphaImageLoader(src='bg.png', sizingMethod='scale')

background:url(bg.gif) no-repeat

content:'※'

@charset "utf-8";
들여쓰기

CSS 코드를 작성할 때는 들여쓰기를 하지 않음.
단, @media를 사용하는 경우와 같이 중괄호가 중첩되는 경우에는 1탭 들여 쓰기로 함.

공백
  • 속성 간 공백 제거
  • 중괄호 좌우 공백 제거
.class p{color:#000;line-height:18px}
빈 줄, 줄바꿈
  • 의미 또는 관련 있는 단락 사이를 구분할 목적으로 빈 줄을 넣는경우, 1줄씩 빈 줄로 함.
    .class p{color:#000;line-height:18px}
  • 속성은 개행하지 않음.
    단, @media 작성시 줄바꿈 한 후, 1탭 들여쓰기 함.
    @media print{
        #header{display:none}
    }
인코딩

반드시 문서 첫 줄에 @charset 선언을 하고 파일 인코딩도 동일한 것으로 선택하여 설정해 준다.

선택자
  • 일반적인 작성에 공통 선택자 '*'를 사용은 지양.
  • id선택자의 범위는 레이아웃 class사용보다 효율적인 경우로 함.
  • 다중선택자 설정은 4개 이하로 함.

[4] !important의 사용 제한

무분별한 사용은 유사 요소의 스타일을 유기적으로 상속 및 재설정 할 수 없게하여 소스의 통일성, 간결성, 효율성을 저해. 콘텐츠 스타일소스 작성에서의 일반적인 사용은 제한.

[5] 타겟 환경 고려

대체 텍스트 제공
  • title과는 그 쓰임이 다르고, 그래픽(이미지)접근성에서의 우선순위는 alt가 더 높으므로 반드시 alt로 제공함.
  • 텍스트 이미지의 경우 : 이미지의 텍스트와 동일한 문구를 alt값에 삽입.
  • 사진 또는 장면의 경우 : alt="○○를 나타내는 자료 화면", alt="○○의 장면" 으로 설명 문구 통일하여 제공함.
  • 약도 또는 그래프의 이미지에 대한 alt속성의 경우 : 설명만으로도 전체의 내용을 파악할 수 있도록 직관적이고 간결하게 내용을 정리하여 제공.
  • 텍스트 아닌 콘텐츠에 대한 대체 텍스트는 그 의미나 기능을 동등한 수준으로 짧고 명확하게 제공.
동영상 콘텐츠의 대체 콘텐츠 제공
  • 최소한 다음 중 한가지 선택 제공.
    • 1. 화면과 동기화된 자막 또는 수화 제공
    • 2. 대본 제공
콘텐츠 검색과 제어
  • 순차검색과 임의검색이 모두 가능해야 함.
  • 모바일 기기의 모든 컨트롤은 누르기 동작으로 제어 가능해야 함.
초점
  • 논리적으로 이동해야 함 : 좌→우, 상→하.
  • 시각적으로 구별되어야 함: css에서 outline속성을 '0'과 'none'으로 설정하지 않는다. 또, js에서 blur처리하지 말아야 함.
  • 클릭이나 터치하는 부분에만 정확히 초점이 이동 될 수 있도록 처리해야 함. 예) 모바일 화면에서, 상단의 전체보기 버튼을 터치해서 레이어를 노출시켰다가 닫기버튼이나 'dim'레이어의 일부를 터치하면 다시 감추는 기능을 구현하는 경우 : 닫기버튼과 'dim'을 묶어 한개의 이벤트로 처리하기 위해 닫기 버튼을 배경처리하는경우 버튼에 해당하지 않는 영역에 까지 초점이 생기는 결과가 되어 바람직하지 않으므로 초점이 생겨야 하는 부분에 영역을 정확히 설정하도록 구현해야 한다.
  • 표시되는 초점의 영역은 콘텐츠의 위치와 크기가 맞도록 제공.
콘텐츠 제어

<iframe name="actionframe" width="0" height="0" title="프로그램처리용 빈 프레이입니다."></iframe>

<iframe name="actionframe" width="0" height="0" title="무배당 변액보험 광고 동영상"></iframe>

숨김처리
  • 의미가 있는 컨텐츠를 영역 배경으로 처리하거나 가상선택자를 사용하여 표현하는 방식의 구현은 하지 않아야 함.
  • 반응형구현의 경우, pc웹과 모바일웹앱을 원소스 구현을 해야 하기 때문에 접근성 처리는 pc와 모바일 모두 적용할 수 있는 방식으로 처리하는 것을 원칙으로 함.
  • ※ 대체 콘텐츠의 처리와 IR구현시 숨김처리의 주의 사항:
    • visibility:hidden, display:none, width:0, height:0, font-size:0, line-height:0 속성의 이와 같은 설정은 하지 말아야 함.

    • 1. 화면밖으로 내보내어 감추기 위해 position:absolute에 top또는 left의 속성값으로 큰 음수를 설정하는 것

      2. text-indent의 속성값으로 큰 음수를 설정하는 것

      이 두가지 방식은 로딩시 문서크기를 크게 계산하여 속도에 영향을 미치고, 모바일 디바이스에서 포커스를 화면에서 잃어버리게 하므로 사용하지 않음.
